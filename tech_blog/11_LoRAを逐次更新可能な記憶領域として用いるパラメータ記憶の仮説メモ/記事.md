# LoRAを逐次更新可能な記憶領域として用いるパラメータ記憶の仮説メモ

### 1. はじめに

LLM エージェントの記憶は、これまで主に外部メモリとして実装されてきた。RAG、ベクトルデータベース、会話ログ、要約メモ、スクラッチパッドなどがその典型である。この設計は、coding agent のように主要な入力がコード、仕様書、issue、ドキュメントなどのテキストである場合には合理的である。必要な情報を検索し、その都度プロンプトへ注入すればよいからである。

しかし、GUI agent や physical AI まで視野を広げると事情は変わる。そこでは観測の中心がテキストではなく、連続した視覚入力、すなわち画像列や動画列になる。全観測をそのまま保存して検索し続ける設計は、計算量、I/O、検索性の面で急速に不利になる。したがって、エージェント記憶は「外部から毎回検索して読むもの」だけでなく、「頻繁に再利用される構造をモデル側へ圧縮して持つもの」としても考える必要がある。

近年の video generation model や world model は、この方向性を示唆している。これらのモデルは、膨大な映像分布や環境ダイナミクスの一部をパラメータの中に圧縮して保持し、将来予測や行動選択に利用しているように見える。もちろん、すべての記憶をパラメータに押し込めるわけではない。しかし少なくとも、**繰り返し使われる知識や環境 priors は、毎回 retrieval するよりも parametric に固定化した方が有利な場合がある**。

本稿では、この観点から次の研究仮説を検討する。

> **継続的な経験列から形成されるエージェント記憶は、外部メモリだけでなく、逐次更新可能な LoRA 状態として表現できるのではないか。**

ただし、ここで関心があるのは単発の text-to-LoRA ではない。焦点は、**既に記憶を担っている学習済み LoRA に対し、未知ドキュメントを追加学習した結果として得られる「更新後 LoRA」を、再学習なしで直接生成する**ことである。つまり本稿の主題は、

$$
(\text{学習済み LoRA}, \text{未知ドキュメント}) \mapsto \text{更新後 LoRA}
$$

という **状態付きの継続学習写像** にある。

---

### 2. 背景

エージェント記憶の実装は大きく二つに分けられる。

1. **外部記憶**
   ベクトル検索、ログ、知識グラフ、要約メモなどとして保存し、必要時にプロンプトへ注入する。

2. **パラメータ記憶**
   事前学習や fine-tuning により、知識やスキルをモデル重みに内在化する。

前者は更新容易性、可逆性、出典性に優れる一方で、検索と再注入のコストがかかる。後者は高速かつ圧縮率が高いが、更新や削除が難しい。この二項対立自体はよく知られているが、近年はその中間に位置する手法が増えている。すなわち、**文書やタスク記述からその場で adapter / LoRA を生成し、ベースモデルに差し込む**という方向である。

この流れにはいくつかの系譜がある。LoRA は、知識や振る舞いの追加分を低ランク差分として分離できる parameter-efficient fine-tuning 手法として、その基盤を提供した。Hypter や HyperLoRA は、タスク記述やテキスト表現から task-specific adapter / LoRA を生成する hypernetwork を提案し、「自然言語からパラメータ更新へ写像する」という発想を押し進めた。さらに Text-to-LoRA は、自然言語のタスク記述を入力として単一の forward pass で LoRA を生成する枠組みを提示した。

また、文書そのものを LoRA に圧縮する研究も現れている。Doc-to-LoRA (D2L) や SHINE は、未見の文書や長文コンテキストを単発で LoRA に変換し、その後の質問応答では元文書を再投入せずに済むようにする。これは、短期コンテキストの一部をパラメータ記憶へ移す試みと解釈できる。

さらに COMPAS は、複数のコンテキストから adapter を生成し、それらをパラメータ空間で代数的に合成する枠組みを提案した。この方向は、

$$
\text{context A} \mapsto L_A,\quad \text{context B} \mapsto L_B,\quad L_A + L_B
$$

という発想を正当化するうえで重要である。

しかし、これらの研究の多くは **単発の文脈圧縮**、**複数文脈の合成**、あるいは **文書からの初期 LoRA 生成** に焦点を当てている。これに対し本稿が関心を持つのは、**既に知識を担っている学習済み LoRA を初期状態として持ち、それに未知ドキュメントを追加学習したときに得られる更新後 LoRA を、毎回の再学習なしに直接近似できるか**という問題である。

すなわち、問いは単なる

$$
T \mapsto L
$$

ではなく、

$$
(L_t, D_{new}) \mapsto L_{t+1}
$$

である。ここで (L_t) は現在の記憶状態を担う学習済み LoRA、(D_{new}) は新たに到着した未知ドキュメント、(L_{t+1}) は **旧知識をできるだけ保持しつつ、新知識を取り込んだ更新後 LoRA** である。

---

### 3. 問題設定

本稿で考えるのは、経験列

$$
D_1, D_2, \dots, D_t
$$

を受け取るエージェントが、外部メモリのみならず、低ランク adapter 状態

$$
L_t
$$

を内部記憶として持つ設定である。ここで (L_t) は、「時刻 (t) までの経験のうち、将来の推論や行動に有用な成分を圧縮した parametric state」とみなす。

重要なのは、本稿では (L_t) を単に「ある文書に対応する LoRA」とはみなさないことである。むしろ (L_t) は、**それまでに蓄積された知識や行動 priors の一部を担う継続的な記憶状態**である。

このとき、新しい未知ドキュメント (D_{new}) が到着したとする。通常であれば、現在の LoRA (L_t) を初期値として追加学習を行い、更新後の LoRA (L_{t+1}) を得ることになる。しかしこの手続きは、新しいドキュメントが来るたびに fine-tuning を要するため、計算コストが高く、継続運用には不向きである。

そこで本稿では、この更新過程そのものを近似する生成器

$$
G(L_t, D_{new}) \approx L_{t+1}
$$

を考える。ここで右辺の (L_{t+1}) は、単なる新規 LoRA ではない。**現在の LoRA (L_t) を初期値にし、未知ドキュメント (D_{new}) に適応しつつ、既存知識の保持性能も満たすよう設計された継続学習手続きの結果**である。

したがって、本稿の問題は「文書を LoRA に変換すること」ではなく、**LoRA の継続学習結果を直接生成すること**として定式化される。

---

### 4. 中心仮説

本稿の中心仮説は三つある。

#### 仮説 1: 頻繁に再利用される記憶は、外部メモリよりも LoRA 状態として内在化した方が効率的である

すべての情報を外部ストアとして保持し、必要時に検索・注入する方式は柔軟だが、長大な履歴や高頻度利用には不向きである。特に長期的に反復使用される知識、行動パターン、環境 priors は、LoRA のような差分パラメータとして保持した方が、レイテンシ、KV-cache、再注入コストの点で有利である可能性がある。

#### 仮説 2: 学習済み LoRA と未知ドキュメントから、継続学習後の更新 LoRA を直接生成する写像を学べる

既存研究の多くは

$$
T \mapsto L
$$

を扱うが、本稿が重視するのは

$$
(L_t, D_{new}) \mapsto L_{t+1}
$$

である。ここで (L_{t+1}) は、新しい未知ドキュメントに適応しつつ、旧知識をできるだけ保持した更新後 LoRA である。もしこの写像を十分よく学習できるなら、未知ドキュメントが到着するたびに追加学習を回さずとも、**継続学習後の状態を単発の生成で近似**できるようになる。

#### 仮説 3: 更新を長く繰り返すと、保持強度の勾配が形成される可能性がある

ここでいう保持強度とは、ある記憶項目が時間経過や追加更新に対してどれだけ残るかを意味する。もし更新則が、再遭遇頻度、利用頻度、重要度、他記憶との結びつきに応じて異なる残存率を自然に生むなら、結果として一部の記憶は長く残り、一部は早く消える。この現象は、明示的な STM/LTM 分離を設計しなくても、**記憶の寿命に関するグラデーション**が emergent に形成される可能性を示す。

ただし、これは現時点ではあくまで探索仮説である。実際には単なる干渉や破壊的忘却に終わるかもしれず、経験的検証が不可欠である。

---

### 5. 提案方向

本稿で想定する研究の最小構成は次の通りである。

#### Step 1: 初期記憶状態としての学習済み LoRA を用意する

まず、ある知識集合や経験集合に対応する学習済み LoRA (L_t) を用意する。これは通常の fine-tuning によって得てもよいし、既存の doc-to-LoRA 系手法やタスク適応済み LoRA を初期記憶状態として採用してもよい。

ここで重要なのは、この LoRA が単発タスクの adapter ではなく、**すでに何らかの知識を担っている継続的記憶状態のスナップショット**として扱われることである。

#### Step 2: 未知ドキュメントで追加学習した更新後 LoRA を教師として構築する

次に、新たな未知ドキュメント (D_{new}) を与え、現在の LoRA (L_t) を初期値として追加学習を行う。ここで得られる更新後 LoRA を (L_{t+1}^{*}) とする。

ただし、この (L_{t+1}^{*}) は単に新ドキュメントに overfit したものではなく、**旧知識の保持性能も考慮した継続学習後の理想状態**である必要がある。したがって教師生成時には、必要に応じて replay、distillation、retention regularization、weight anchoring などを導入し、**旧知識の保持と新知識の獲得を両立した更新後 LoRA** を作る。

#### Step 3: 更新後 LoRA を直接生成する更新器を学習する

本稿の中心はここにある。すなわち、

$$
G(L_t, D_{new}) \approx L_{t+1}^{*}
$$

を学習し、現在の記憶状態と未知ドキュメントから、追加学習後に到達するべき LoRA を直接生成できるようにする。

このとき (G) は、単に新ドキュメントを LoRA に変換するモデルではない。むしろ、

* 既存知識をどこまで維持すべきか
* 新知識をどこへどの程度書き込むべきか
* 競合する知識をどう解決するか
* 何を残し、何を弱めるか

といった **継続学習の更新ダイナミクス** を近似するモデルである。

#### Step 4: 繰り返し更新下での保持構造を観測する

更新器を一回限りで使うのではなく、繰り返し適用することで

$$
L_0 \to L_1 \to L_2 \to \cdots
$$

という系列を構成する。このとき、どの記憶が残り、どの記憶が消えるかを追跡する。特に、

* 反復頻度
* 再遭遇頻度
* 利用頻度
* 時間間隔
* 他記憶との結合度

といった要因が、残存率にどのような影響を与えるかを測定する。ここで保持強度の非一様な分布が生じるなら、それを STM/LTM 的分化の前段階として解釈できるかもしれない。

---

### 6. なぜこの方向が必要か

coding agent では、外部メモリによる検索と再注入だけでもかなりの性能が得られる。これは、主データがテキストであり、exact match や局所参照が重要だからである。しかし GUI agent や physical AI では、観測が高頻度・高次元の視覚列であり、全履歴を毎回検索して再利用する設計はスケールしにくい。

このとき、エージェントに必要なのは、生の全履歴そのものではなく、反復パターン、環境 priors、操作手順、因果構造のような再利用可能な圧縮表現である。もしこれらの一部を LoRA 状態として固定化できるなら、外部記憶に依存しすぎない軽量な推論ループが可能になる。

その意味で、理想的な記憶設計は外部メモリと parametric memory の二者択一ではない。むしろ、

1. 新情報をまず外部メモリに保存する
2. 繰り返し参照・使用される
3. 要約・抽象化される
4. LoRA 状態として固定化される
5. 必要ならさらにベースモデルへ蒸留される

という多段階の固定化パイプラインが有望である。本稿はそのうち、特に **「現在の LoRA 記憶状態に対して未知ドキュメントを統合した更新後 LoRA を、再学習なしで直接得る」** という段階に注目する。

---

### 7. 想定される利点

この方向が成立するなら、少なくとも以下の利点が期待できる。

* 新しいドキュメントが到着するたびに LoRA を追加学習しなくても、**更新後 LoRA を直接生成**できる
* 長大な履歴を毎回再投入せずに済むため、推論レイテンシと KV-cache コストを削減できる
* 頻繁に使われる知識を parametric に固定化できる
* 新しい経験を逐次的に継ぎ足す継続記憶に近づける
* 単純な STM/LTM 分離ではなく、保持強度のグラデーションを観測対象にできる
* GUI / physical AI のように視覚中心の観測でも、重要な再利用構造を圧縮して扱いやすくできる

特に重要なのは、LoRA を単発で生成するのではなく、**既にある知識を保ったまま新知識を統合した更新状態を直接近似する**点である。これにより、text-to-LoRA を超えて **continual adaptation as generation** という視点が得られる。

---

### 8. 限界と注意点

ただし、この方向には明確な限界がある。

第一に、LoRA 状態は exact retrieval に向かない。契約書の文面、画面上の具体的位置、監査のための出典などは、外部メモリの方が適している。したがって、LoRA 記憶は外部メモリの完全な代替ではなく、補完的な圧縮記憶として捉えるべきである。

第二に、削除や修正が難しい。LoRA はベースモデルよりは分離可能だが、それでもどの知識がどこに保存されたかは不透明であり、選択的忘却や訂正は自明ではない。

第三に、教師となる更新後 LoRA 自体の定義が難しい。単に未知ドキュメントで further-tuning しただけでは、旧知識が壊れてしまう可能性がある。したがって、「旧知識保持と新知識獲得を両立した更新後 LoRA」をどう構成するかは、本研究の前提として重要な設計課題である。

第四に、更新干渉の問題がある。新しい経験を取り込むたびに、既存記憶が壊れる可能性がある。特に、継続学習の結果を生成で近似するという設定では、**生成器自身が forgetting や interference を増幅する**可能性もある。

第五に、保持強度のグラデーションが本当に emergence として観測されるかは未検証である。頻度依存のきれいな分化が生じる可能性もあるが、単に一部の情報が壊れにくいだけ、あるいは逆に無秩序に劣化するだけという可能性もある。

したがって、本稿の提案は「LoRA が人間的記憶をそのまま実現する」という主張ではなく、**継続学習後の LoRA 状態を直接生成することで、逐次記憶の最小形式がどこまで実現できるかを問う試み**として位置づけるべきである。

---

### 9. 検証のための評価観点

この研究方向を成立させるには、少なくとも以下の評価軸が必要である。

* **Retention**: 旧知識をどれだけ保持できるか
* **Acquisition**: 新しい未知ドキュメントの知識をどれだけ正しく取り込めるか
* **Update approximation quality**: 直接生成した更新後 LoRA が、実際の継続学習後 LoRA をどれだけ近似できるか
* **Interference**: 更新により無関係な既存知識がどれだけ壊れるか
* **Conflict handling**: 矛盾する情報が来たときに、上書き・共存・条件付き保持のいずれが起きるか
* **Forgetting profile**: 頻度、再遭遇、時間間隔に応じて残存率がどう変化するか
* **Efficiency**: 実際に毎回 LoRA を追加学習する場合、long-context を毎回再投入する場合、RAG を用いる場合と比べて、計算量、メモリ使用量、レイテンシがどう変わるか

これらを分離して測定しない限り、「更新後 LoRA を生成できた」と言っても、それが単に短期タスクでうまくいっただけなのか、真に継続記憶として機能したのかを区別できない。

---

### 10. おわりに

本稿の主張は単純である。エージェント記憶は、外部メモリだけでなく、逐次更新可能な parametric state としても捉えられるべきではないか、ということである。特に本稿が重視するのは、

$$
(L_t, D_{new}) \mapsto L_{t+1}
$$

という状態付き更新であり、しかも (L_{t+1}) を単なる新規 LoRA ではなく、**既存知識をできるだけ保持しつつ未知ドキュメントを取り込んだ継続学習後の状態**として定義する点にある。

もしこの更新写像を十分よく学習できるなら、エージェントは未知ドキュメントが到着するたびに高価な追加学習を実行しなくても、**「追加学習したなら到達していたはずの LoRA 状態」** を直接得られるかもしれない。これは、LoRA を単発の適応手段としてではなく、継続的に更新される記憶状態として扱うための重要な一歩になる。

この方向の重要性は、長文テキスト処理そのものよりも、むしろ GUI agent や physical AI のような高頻度・高次元観測の文脈で大きくなる。そこでは、すべての履歴を外部から毎回読み直すのではなく、反復的に利用される構造を圧縮し、行動に有用な形で固定化することがより重要になるからである。

もちろん、この方向には限界がある。LoRA は exact retrieval を代替しないし、更新干渉や削除困難の問題も残る。しかしそれでも、外部メモリ一辺倒ではなく、**継続学習後の更新 LoRA を直接生成する逐次記憶モデル**として LoRA 状態を扱うことには十分な研究価値がある。

さらに興味深いのは、その更新過程を長く回したとき、保持強度のグラデーションが自然に形成されるかもしれない点である。もしそうなら、短期記憶と長期記憶は設計された別モジュールとしてではなく、更新ダイナミクスの結果として現れる。これは、エージェント記憶の設計を「どの箱に何を入れるか」から、「どのような更新則がどのような記憶構造を生むか」へと転換する視点を与える。
